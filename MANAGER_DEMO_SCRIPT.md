# TaskFlow Manager Demo Script

## 1. Project Introduction

Hello, this is my TaskFlow full-stack task manager application.

It includes:
- React frontend
- Node.js and Express backend
- MySQL database
- Sequelize ORM
- JWT authentication
- Kanban-style task management

## 2. Frontend UI

Start by showing the login page.

Explain:
- Users can login or register.
- After login, the JWT token is stored in localStorage.
- API requests automatically attach the token.

Then show the dashboard.

Explain:
- The top area shows the logged-in user.
- The dashboard says "Welcome back" with the user's name.
- Metrics show total tasks, pending tasks, under review tasks, and completed tasks.
- Tasks are organized into four columns:
  - To Do
  - In Progress
  - Under Review
  - Completed
- Users can search tasks.
- Users can filter tasks by priority.
- Users can create, edit, move, and delete tasks.

## 3. Frontend Code

Open:
frontend/src/App.jsx

Explain:
- App.jsx controls authentication state, task state, filters, modals, and dashboard rendering.
- It uses React hooks like useState, useEffect, and useMemo.
- It calls the backend through Axios.

Open:
frontend/src/api.js

Explain:
- Axios base URL is configured here.
- The request interceptor adds the JWT token as:
  Authorization: Bearer token
- If the token is invalid, localStorage is cleared.

Open:
frontend/src/styles.css

Explain:
- Tailwind CSS is used for the UI styling.
- Shared button, modal, card, and control styles are defined here.

## 4. Backend Code

Open:
backend/server.js

Explain:
- Express server is created here.
- CORS and JSON middleware are enabled.
- Auth routes are mounted at /api/auth.
- Task routes are mounted at /api/tasks.
- Sequelize connects to MySQL and syncs the tables.

Open:
backend/routes/authRoutes.js

Explain:
- POST /api/auth/register creates a new user.
- POST /api/auth/login logs in a user.
- GET /api/auth/me returns the current logged-in user.
- Validation is handled using express-validator.

Open:
backend/routes/taskRoutes.js

Explain:
- All task routes are protected by JWT middleware.
- GET /api/tasks fetches tasks.
- POST /api/tasks creates a task.
- PUT /api/tasks/:id updates a task.
- DELETE /api/tasks/:id deletes a task.

Open:
backend/controllers/authController.js

Explain:
- Register checks if email already exists.
- Password is hashed before saving.
- Login compares password using bcrypt.
- On success, backend returns user data and JWT token.

Open:
backend/controllers/taskController.js

Explain:
- Tasks are always filtered by logged-in user ID.
- This prevents one user from seeing another user's tasks.
- The controller also calculates dashboard stats.

## 5. Database

Open:
backend/config/database.js

Explain:
- Sequelize connects to MySQL using environment variables from backend/.env.
- Database name is task_manager.

Open:
backend/models/User.js

Table: users

Columns:
- id: integer primary key, auto increment
- name: string, required
- email: string, required and unique
- password: string, required, stored as bcrypt hash
- createdAt: timestamp generated by Sequelize
- updatedAt: timestamp generated by Sequelize

Important:
- Password hashing happens in beforeCreate and beforeUpdate hooks.
- comparePassword checks login password with bcrypt.

Open:
backend/models/Task.js

Table: tasks

Columns:
- id: integer primary key, auto increment
- title: string, required
- description: text, optional
- status: enum: todo, in-progress, under-review, completed
- priority: enum: low, medium, high
- dueDate: date only, optional
- userId: foreign key to users.id
- createdAt: timestamp generated by Sequelize
- updatedAt: timestamp generated by Sequelize

Open:
backend/models/index.js

Explain:
- One user has many tasks.
- One task belongs to one user.
- If a user is deleted, their tasks are deleted using cascade.
